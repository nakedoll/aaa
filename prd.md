## 1. 제품 개요

### 1-1. 목적
한국콘텐츠기업협회의 기업 회원 정보와 연간 회비 납부 현황을 효율적으로 관리하기 위한 크로스플랫폼 데스크탑 애플리케이션을 개발한다. Windows(.exe)와 macOS(.dmg) 모두 지원한다.

### 1-2. 배경
- 회원 수가 지속적으로 증가하고 있어 체계적인 관리 도구가 필요
- 관리자 1인이 단독으로 사용하는 내부 도구
- 외부 서버 없이 로컬에서 안전하게 데이터 보관

### 1-3. 목표 사용자
- 단일 관리자 (협회 운영 담당자 1인)

### 1-4. 범위
- 회원(기업) 정보 등록 및 관리
- 연간 회비 납부 현황 관리
- 통계 대시보드
- 데이터 자동 백업

## 3. 데이터베이스 설계

### 3-1. members (회원 테이블)

| 컬럼명 | 타입 | 제약 | 설명 |
|---|---|---|---|
| id | INTEGER | PK, AUTOINCREMENT | 고유 식별자 |
| company_name | TEXT | NOT NULL | 회사명 |
| ceo_name | TEXT | NOT NULL | 대표자 이름 |
| contact_name | TEXT | | 담당자 이름 |
| contact_phone | TEXT | | 담당자 연락처 |
| contact_email | TEXT | | 담당자 이메일 |
| joined_at | DATE | NOT NULL | 가입일 |
| status | TEXT | NOT NULL, DEFAULT 'active' | 회원 상태 (active / dormant / withdrawn) |
| memo | TEXT | | 메모 / 특이사항 |
| created_at | DATETIME | NOT NULL | 등록일시 |
| updated_at | DATETIME | NOT NULL | 수정일시 |

### 3-2. dues (회비 납부 테이블)

| 컬럼명 | 타입 | 제약 | 설명 |
|---|---|---|---|
| id | INTEGER | PK, AUTOINCREMENT | 고유 식별자 |
| member_id | INTEGER | FK → members.id | 회원 외래키 |
| year | INTEGER | NOT NULL | 납부 연도 |
| amount | INTEGER | NOT NULL | 납부 금액 |
| paid_at | DATE | | 납부일 |
| status | TEXT | NOT NULL, DEFAULT 'unpaid' | 납부완료(paid) / 미납(unpaid) |
| created_at | DATETIME | NOT NULL | 등록일시 |
| updated_at | DATETIME | NOT NULL | 수정일시 |

- UNIQUE 제약: (member_id, year)
- 납부는 전액 일시 납부만 허용 (부분 납부 없음)

### 3-3. dues_settings (연도별 기준 회비 테이블)

| 컬럼명 | 타입 | 제약 | 설명 |
|---|---|---|---|
| year | INTEGER | PK | 연도 |
| base_amount | INTEGER | NOT NULL | 해당 연도 기준 회비 금액 |
| created_at | DATETIME | NOT NULL | 등록일시 |
| updated_at | DATETIME | NOT NULL | 수정일시 |

- 기준 회비는 연도별로 변경 가능
- 모든 기업 회원의 회비 금액은 동일

## 4. 기능 요구사항

### 4-1. 통계 대시보드 (메인 화면)

**목적**: 앱 실행 시 현황을 한눈에 파악

**stat 카드 4종** (좌→우 순서):

| 카드 | 표시값 | 서브텍스트 | 테두리 색 |
|---|---|---|---|
| 전체 회원 | 총 회원 수 | 활동 N · 휴면 N · 탈퇴 N | 파란색(accent) |
| 납부 완료 | 납부 완료 수 | 납부율 N% | 초록색 |
| 미납 | 미납 회원 수 | "활동 회원 기준" 고정 문구 | 황색(amber) |
| 기준 회비 | 현재 연도 기준 회비 금액 (원) | "전년 대비 동일" 등 | 회색(slate) |

- stat 카드는 클릭 시 회원 목록으로 이동 (기준 회비 카드는 클릭 불가)
- 카드 진입 애니메이션: `fadeUp` (각 카드 0.07s 간격 딜레이)

**차트 2종** (카드 아래, 2열 그리드):

1. **기간별 회원 가입 현황** (막대 차트, Chart.js)
   - 탭: "월별" (기본) / "연도별" 전환
   - 월별: 현재 연도의 1–12월 신규 가입 수
   - 연도별: 전체 연도별 신규 가입 수 (색상 투명도로 연도 구분)
   - 차트 높이 200px 고정

2. **연도별 수금 추이** (라인 차트, Chart.js)
   - 탭: "금액" (기본) / "납부율" 전환
   - 금액: 연도별 수금액 합계 (만원 단위), 초록색 라인
   - 납부율: 연도별 납부율 (%), 황색(amber) 라인
   - 차트 높이 200px 고정

- 차트 카드 진입 애니메이션: `fadeIn` 0.3s 딜레이

**하단 2열 테이블**:

- **최근 납부 현황** (좌): 회사명 / 납부일 / 금액 / 상태 — 최근 5건
- **미납 회원 현황** (우): 회사명 / 가입일 / 회원 상태 — 5건 미리보기, 우상단에 미납 기업 수 표시

**동작**:
- 현재 연도를 기준으로 자동 계산
- stat 카드 클릭 시 해당 조건으로 필터된 회원 목록으로 이동
- 테이블 행 클릭 시 해당 회원 상세 화면으로 이동

### 4-2. 회원 목록

**목적**: 전체 회원을 테이블 형식으로 조회

**표시 열** (좌→우 순서):
- 회사명, 대표자, 담당자, 담당자 연락처, **가입일**, 회원 상태, 현재 연도 납부 상태

  > ⚠️ PRD v1.0 대비 변경: "가입일" 열 추가됨 (목업 기준)

**동작**:
- 기본 정렬: 회사명 가나다순
- 행 클릭 시 회원 상세 화면으로 이동
- 우상단 "회원 추가" 버튼으로 회원 추가 화면으로 이동
- 페이지네이션: 10건/페이지

### 4-3. 회원 검색 및 필터

**조건** (복합 적용 가능):

| 필터 | UI 형태 | 옵션 |

### 4-4. 회원 상세 화면

**목적**: 회원 정보 전체 조회 및 편집

**표시 내용**:
- 회원 기본 정보 전체
- 연도별 회비 납부 이력 목록

**동작**:
- 정보 수정 후 저장 버튼 클릭으로 확정
- 탈퇴 처리 버튼 → 확인 팝업("탈퇴 처리하시겠습니까?") 후 실행
- 탈퇴 처리 시 데이터는 삭제되지 않고 status = 'withdrawn'으로 변경

### 4-5. 회원 추가

**입력 항목**:

| 항목 | 필수 여부 |
|---|---|
| 회사명 | 필수 |
| 대표자 이름 | 필수 |
| 담당자 이름 | 선택 |
| 담당자 연락처 | 선택 |
| 담당자 이메일 | 선택 |
| 가입일 | 필수 |
| 회원 상태 | 필수 (기본값: 활동) |

**유효성 검사**:
- 회사명: 공백 불가
- 담당자 이메일: 형식 검사 (입력 시)
- 담당자 연락처: 숫자/하이픈만 허용 (입력 시)

---

### 4-6. 회비 관리

**회비 등록**:
- 회원 상세 화면에서 연도별 납부 등록
- 납부 연도, 납부 금액, 납부일 입력
- 동일 회원의 동일 연도 중복 등록 불가

**기준 회비 설정**:
- 설정 화면에서 연도별 기준 회비 금액 등록 및 수정
- 회비 등록 시 해당 연도의 기준금액을 자동으로 불러옴 (수정 가능)

**미납 처리**:
- 기준 회비가 설정된 연도에 납부 기록이 없는 회원을 미납으로 간주
- 미납 회원 목록 별도 조회 가능

---

### 4-7. 백업 및 복원

**자동 백업**:
- 앱 실행 시마다 SQLite DB 파일을 자동 복사
- 백업 파일명: `backup_YYYY-MM-DD_HH-MM-SS.db`
- 최근 5개 보관, 초과 시 가장 오래된 파일 자동 삭제
- 백업 저장 위치: userData 디렉터리 하위 `backups/` 폴더

**복원**:
- 설정 화면에서 백업 파일 목록 조회 가능
- 선택한 백업 파일로 복원 → 확인 팝업 후 실행
- 복원 후 앱 재시작

## 5. 비기능 요구사항

### 5-1. 보안
- `nodeIntegration: false`, `contextIsolation: true` 적용
- SQLite 접근은 메인 프로세스에서만 처리
- IPC 채널을 통해 렌더러 ↔ 메인 통신
- 모든 SQL 쿼리는 파라미터 바인딩 사용 (SQL 인젝션 방지)

### 5-2. 성능
- 회원 100개 기준 목록 로딩 1초 이내
- 검색/필터 결과 즉시 반영

| `settings-dues` | 기준 회비 설정 | 연도별 기준 회비 금액 등록 및 수정 | 설정 |
| `settings-backup` | 백업 / 복원 | 백업 파일 목록 조회 및 복원 | 설정 |

## 8. 배포

| 항목 | Windows | macOS |
|---|---|---|
| 배포 형식 | NSIS Installer (.exe) | DMG |
| 지원 아키텍처 | x64 | Apple Silicon (arm64) + Intel (x64) 유니버설 |
| 코드 서명 | 선택 사항 (없어도 실행 가능, SmartScreen 경고 발생) | Apple Developer ID 서명 + 공증 권장 |
| 빌드 도구 | electron-builder | electron-builder |

**Windows 배포 시 유의 사항**:
- 서명 없이 배포 시 Windows SmartScreen에서 "알 수 없는 앱" 경고가 표시됨 — 수신자가 "추가 정보 → 실행" 클릭으로 우회 가능
- `better-sqlite3`는 native 모듈이므로 빌드 전 `electron-rebuild` 또는 `prebuild-install` 실행 필요
- DB 저장 경로 (Windows): `C:\Users\[사용자]\AppData\Roaming\[앱이름]\db\members.db` — `app.getPath('userData')` 사용 시 자동 처리

**macOS 배포 시 유의 사항**:
- DB 저장 경로 (macOS): `~/Library/Application Support/[앱이름]/db/members.db`
- Apple Developer Program 가입 필요 (연 $99) — 서명/공증 없이도 개발 환경에서 실행 가능
